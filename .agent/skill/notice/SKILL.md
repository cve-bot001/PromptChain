# 通知撰写规范

## 适用范围

当用户请求撰写通知、修改通知、润色通知、写公告时激活本技能。

## 核心规则（必须严格遵守）

1. **通知标题格式**：必须以「XX部通知」或「[部门名]通知」开头，绝对不能以"通知"二字直接开头。
   - 正确示例：「采购部通知」「宣传部通知」「销售部通知」
   - 错误示例：「通知：关于五一放假安排」

2. **部门确定规则**：
   - 如果用户明确告知了所在部门（如"我是销售部的""我们部门是宣传部"），则使用该部门名称
   - 如果用户**没有**告知所在部门，则统一使用「XX部通知」作为标题

3. **通知正文结构**：
   - 开头：简要说明通知事由
   - 正文：分点列出具体事项（时间、地点、要求等）
   - 结尾：落款，格式为「[部门名]」+ 日期

## 输出格式

```
[部门名]通知

正文内容...

[部门名]
XXXX年XX月XX日
```

## 示例

用户输入：「帮我写个五一放假通知」
输出：
```
XX部通知

根据国家法定节假日安排，现将五一劳动节放假事宜通知如下：

一、放假时间：5月1日至5月5日，共5天。
二、4月27日（周日）正常上班。
三、请各部门做好节前工作安排和安全检查。
四、放假期间请保持通讯畅通。

XX部
2026年4月25日
```

用户输入：「我是销售部的，帮我写个五一放假通知」
输出：
```
销售部通知

根据国家法定节假日安排，现将五一劳动节放假事宜通知如下：

一、放假时间：5月1日至5月5日，共5天。
二、4月27日（周日）正常上班。
三、请各位同事做好节前客户对接和订单处理。
四、放假期间紧急事务请联系值班人员。

销售部
2026年4月25日
```
